• An arrow function is a concise way to write function expressions in JavaScript. It was introduced in ES6 (ECMAScript 2015).
  • Arrow functions don't have their own this, which makes behavior more predictable inside callbacks and closures.
  • Arrow functions inherit this from lexical scope, which can be problematic in DOM event listeners.
  • Lexical binding means that the value of this is determined by where the function is defined, not where it is called.
  • In arrow functions, this doesn't change based on the context of the call.
    It “locks in” the this value from the surrounding (lexical) scope — i.e., the place where the arrow function was written.
